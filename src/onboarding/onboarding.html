an introduction in three acts

Your downloads, paused
on the threshold.

Popy puts every file you download into a sealed sandbox — invisible to your operating system — and renames it so nothing can execute it by accident. It waits there, harmless, until you release it.

I.

A file arrives.

When something tries to download — a PDF, an installer, an image, anything — Popy catches it in mid-air. It streams the bytes into the browser's Origin Private File System, a storage area your OS can't see. The file gets a _popy suffix so even if it somehow slipped out, nothing would know how to open it.

server → Popy sandbox ⊘ your filesystem
II.

Nothing can touch it.

While the file sits in the sandbox, Windows can't index it, macOS can't preview it, your antivirus's on-access scanner can't even see it. The most dangerous thing a file can do — be parsed by a vulnerable operating-system handler the instant it lands — simply cannot happen.

  • ◦ No double-click execution
  • ◦ No thumbnail-parser attacks
  • ◦ No preview-handler exploits
  • ◦ No Spotlight / Windows Search indexing
III.

You decide what happens.

From Popy's dashboard you can look the file over, preview PDFs and images safely in the browser, check its SHA-256 against VirusTotal, then release it to a location you choose — or delete it forever. Releasing strips the _popy suffix; deleting is instant and complete.

You will never again wonder whether that thing you downloaded an hour ago has been quietly parsed by something.

·

Some honesty before you start.

Popy is not an antivirus. It does not scan files for malicious content. What it does is simpler and, in practice, more reliable: it refuses to let the operating system near a downloaded file until you say so.

For a minority of downloads — ones created by JavaScript in the page, ones behind single-use confirmation tokens, ones larger than the sandbox can hold — Popy falls back to a second line of defense: saving the file with a _popy suffix in a dedicated quarantine folder. It's still defused, but it lives on your disk. You'll always see which path was taken.
